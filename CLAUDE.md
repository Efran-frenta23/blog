# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Purpose

Internal documentation site for **Dalang.io** (cloud hosting provider operated by PT Mashanz Sumber Terbuka & PT Komponen Pusat Data). Contains product docs, company info, and office SOPs/tutorials. Content is primarily in **Bahasa Indonesia**.

Built with **Docusaurus 3.7** (React 19). Repo directory is named `blog` but the site has two distinct surfaces — treat them differently:

- `docs/` — structured product & reference documentation (sidebar auto-generated from filesystem).
- `blog/` — long-form SOP-style articles and tutorials (e.g. Linux distro guide, Ubuntu install, genset F-Series manual). Filenames must follow `YYYY-MM-DD-slug.md`.

## Commands

```bash
yarn                # install deps (use yarn, not npm — README + lockfile convention)
yarn start          # local dev server with hot reload
yarn build          # produce static site in build/
yarn serve          # serve the built site
yarn clear          # clear .docusaurus cache (use when config/sidebar changes don't take effect)
yarn deploy         # GH-pages deploy (needs GIT_USER=... or USE_SSH=true)
```

Node >= 18 required.

## Architecture notes that aren't obvious from one file

- **Repurposed template files**: files under `docs/tutorial-basics/` (e.g. `create-a-page.md`, `create-a-document.md`, `create-a-blog-post.md`) keep their **original Docusaurus-template filenames** but their **content has been rewritten** to describe Dalang.io products (VPS, services, deploy tutorial). Don't assume filename reflects content — read the file. Ordering is controlled by `sidebar_position` in frontmatter, not filename.
- **Sidebar is autogenerated** (`sidebars.js` uses `{type: 'autogenerated', dirName: '.'}`), so adding a new markdown file under `docs/` automatically adds it to the sidebar. Use `_category_.json` for section labels and `sidebar_position` frontmatter for ordering within a category.
- **Broken links fail the build**: `onBrokenLinks: 'throw'` in `docusaurus.config.js`. Verify cross-references before committing.
- **`docusaurus.config.js` still contains upstream placeholder values** ("My Site", `organizationName: 'facebook'`, tagline "Dinosaurs are cool", edit URLs pointing at facebook/docusaurus). These are intentionally untouched so far — don't "fix" them as cleanup; only change them if the user asks for site rebranding.
- **Images for blog posts** live in `static/img/` and are referenced as `/img/<name>` (absolute path served from site root), not as relative paths. Example pattern: `genset1.jpeg … genset17.jpeg` pairs with `blog/2026-03-27-tutorial-genset-tdg-15000s.md`.
- **Blog post dates in filenames are future-dated (2026-xx-xx)** — that is intentional for this repo, not a mistake to "correct".
- `blog/authors.yml` and `blog/tags.yml` still contain Docusaurus demo entries (yangshun, slorber, facebook/hola tags). New posts that use authors/tags should add entries here.
